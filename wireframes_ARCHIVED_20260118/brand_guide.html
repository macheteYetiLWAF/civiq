1. Color System

Primary Gradient (Impartiality)

CRITICAL RULE: Blue must ALWAYS appear on the LEFT, Red on the RIGHT. This applies to logo, buttons, progress bars, any political spectrum visualization.
Full Gradient Blue → Purple → Red
Logo Gradient Blue 40% → Red 60% 
/* Logo Gradient */
background: linear-gradient(90deg, #2563EB 40%, #DC2626 60%);
-webkit-background-clip: text;
-webkit-text-fill-color: transparent;

/* Button Gradient (full spectrum) */
background: linear-gradient(90deg, #2563EB 0%, #8B5CF6 50%, #DC2626 100%);

Active State Color

Decision (Jan 4, 2026): Active/selected states use Purple (#8B5CF6) - NOT blue or red alone. Purple represents the blend of both sides, politically neutral.

4. Component Patterns

Buttons

TypeUse CaseStyle
Primary (Gradient) Main CTAs: Sign In, Continue, Start Exploring background: linear-gradient(90deg, #2563EB 0%, #8B5CF6 50%, #DC2626 100%)
Secondary Active filter state, selected tabs background: var(--bg-tertiary)
Outline Secondary actions, inactive filters border: 1px solid var(--border)
NO solid blue or red buttons. Any button that could appear "political" must use the gradient.

Toggle Switches

Active toggle background: #8B5CF6 (Purple)

Inactive toggle background: var(--border)

8. Decision Log

DateDecisionRationale
Jan 4, 2026 Active state = Purple (#8B5CF6) Cannot use solid blue or red - politically charged. Purple is the blend.
Jan 4, 2026 Media icon = Article (fa-file-lines) After exploring 35+ options, Article best represents content consumption.
Jan 4, 2026 Font Awesome only - no custom SVGs Custom line icons looked "like stick figures". FA is professional and consistent.
Jan 4, 2026 Bias Rebalance moved to Media Keep Home feed clean. Bias tracking belongs with reading/sources context.
Jan 4, 2026 3-step onboarding (not 4) Simplified flow: Register → Welcome → Personalize → Home
Jan 4, 2026 Logout button uses full gradient No solid red - any "action" button that could seem political uses gradient.
This is a living document. Update as decisions are made. Keep rejected options documented for context.
